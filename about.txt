при изменении файлов может быть такая проблема, что в браузере закешировались скрипты. и новые изменения не будут приходить

для решения этой проблемы есть еще один паттерн вебпака.
вместо задания конкретного названия - bundle, используем этот паттерн.

было filename: '[name].bundle.js',
стало filename: '[name].[contenthash].js',
при сборке получим новые бандлы.
main.(сгенерированный хеш).js
analytics.(сгенерированный хеш).js

изменим один из скриптов для проверки генерации хеша.
немного исправили файл аналитики.
создался новый бандл для этого файла со своим новым хешем. папку dist все еще не очищаем от старых бандлов

https://webpack.js.org/configuration/output/ - документация описания всех возможностей настройки output
