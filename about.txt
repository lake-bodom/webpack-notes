для удобства при импортировании в js файлах, чтобы не прописывать их расширения, есть настройка для вебпака.

было import WebpackLogo from './assets/webpack-logo.png';
стало import WebpackLogo from './assets/webpack-logo';

добавляем в конфиг файл объект resolve.
в котором прописвыаем необходимые расширения в массив extensions
resolve: {
  extensions: ['.js', '.json', '.png']
},

при этом, не добавляя это правило расширения js и json установлены по умолчанию

для прописывания абсолютных путей до файла, вместо относительных можно добавить alias.
для демонстрации создадим директорию models, в которую поместим файл Post.js
в объекте resolve, который был добавлен уже в конфиг добавлям объект, в котором будут прописаны необходимые алиасы
пропишем для models и папки src
resolve: {
  extensions: ['.js', '.json', '.png'],
  alias: {
    '@models': path.resolve(__dirname, 'src/models'),
    '@': path.resolve(__dirname, 'src'),
  }
},

результат манипуляций
было import Post from './models/Post';
стало import Post from '@models/Post';
