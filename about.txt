на прошлом этапе была проблема с тем, что файл аналитики не собирался вебпаком.

для решения проблемы стоит определить несколько входных файлов приложения.

в конфиге поменяем entry для файлов
entry: {
  main: './src/index.js',
  analytics: './src/analytics.js'
}

теперь после сборки должны получиться два файла - main и analytics.

но так как filename: 'bundle.js', описывает только один выходной файл, сборка пройдет с ошибкой

для этого воспользумся паттерном вебпака для наименования файлов.
filename: '[name].bundle.js'

вебпак подставит сам вместо [name] main и analytics.

сборка успешна. получилось 2 файла main.bundle.js и analytics.bundle.js

исправим названия скриптов при подключении в index.html

при таких манипуляциях если производились сборки, в папке dist останется предыдщий файл бандла. эта проблема будет решена далее
